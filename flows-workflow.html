Flow Production — Granular ClickUp Workflow

Detailed status-by-status flow with roles, decisions, and ClickUp statuses.

PM – Project Manager S – Strategist C – Copywriter D – Designer
Strategy
S · Create flow strategy in FigJam
Present on call or async?
Call
PM · Schedule strategy call
S · Present strategy on call
Async
S · Record Loom walkthrough
PM · Send Loom + FigJam to client
Client approves the strategy?
✓ Yes, no changes
S · Final check on strategy
↓ continue
✓ Yes, small changes
S · Make changes on/after call
↓ continue
✗ No, big changes
S · Make changes
PM · Share via Slack for async approval
↩ Re-send strategy
More than 3 emails in the flow?
✓ Yes
PM · Deploy CU task with nested subtask paths
↓ continue
✗ No
PM · Deploy CU task with regular template
↓ continue
Copy
C · Write copy in Google Doc, add doc link to task, resolve subtask, move status
Copy Review
S · Review copy, bold chosen SL/PT in doc
Strategist approves the copy?
✓ Approved
↓ Move to Design
✗ Edits needed
S · Create "copy edits" subtask
↩ Back to Copy
Design
D · Design in Figma, add Figma link to task, resolve subtask, move status
Design Review
S · Review design in Figma, leave comments for PM on things to highlight
Strategist approves the design?
✓ Approved
S · Tag PM, confirm which emails are ready
↓ Move to Design Approved
✗ Edits needed
S · Create "design edits — see [place]" subtask
↩ Back to Design
Design Approved
PM · Prepare Figma file: duplicate WIP → Client Review pages, strategy page, SL/PT, L&C notes, proofread
S · Review prepared Figma file and sign off
PM · Send Figma to client
In Client Review
Client reviews the flow and leaves feedback as comments in Figma. Client tags PM in ClickUp when done.
Client feedback?
Scenario A — Approved, no changes
PM · Comment approved, tag Strategist. Add subtasks: Build Flow, Upload, Final Review, QA
Client Reviewed
S · Build flow structure in Klaviyo, add flow link to task description
Design
D · Upload approved email & SMS designs to Klaviyo
  • Set up universal blocks
  • Configure dynamic blocks & properties
  • Add all links to blocks and CTAs
  • Preview own work before marking complete
Design Review
S · Final Design Review in Klaviyo
  • Figma changes fully reflected in Klaviyo
  • All links correct
  • Dynamic blocks/properties configured
  • Discount codes & dynamic properties/codes correct
  • A/B tests configured
Design Approved
PM · Add SL/PT to Klaviyo flow
PM · QA: check SL/PT, desktop, mobile, dark mode, links, dynamic properties
PM · Send previews to client & kasey@luckandco.agency
PM · Update FigJam board with Klaviyo preview links
Client Approved
S · Activate flow in Klaviyo (+ A/B tests)
S · Deactivate old flow if replacing
S · Update CU task description ("Activated on [date]")
Activated ✓
Scenario B — Minimal changes
PM · Tag Strategist with feedback. Create subtasks: "Review feedback + take next steps" and "Build Flow in Klaviyo" (both → S)
Client Reviewed
S · Review Figma comments, push back if needed
S · Build flow structure in Klaviyo, add flow link to task
Copy edits needed?
Quick
S · Edits doc directly, marks NEW
Bigger
S · Create "Copy Edits — see Figma"
C · Apply edits → Copy Review
S · Create subtask: "Design Edits + Upload — see [Copy doc &] Figma"
Design
D · Apply Figma edits + upload emails & SMSs to Klaviyo in one step
  • Set up universal blocks
  • Configure dynamic blocks & properties
  • Add all links to blocks and CTAs
  • Preview own work before marking complete
Design Review
S · Final Design Review in Klaviyo
  • Figma changes fully reflected in Klaviyo
  • All links correct
  • Dynamic blocks/properties configured
  • Discount codes & dynamic properties/codes correct
  • A/B tests configured
Design Approved
PM · Add SL/PT to Klaviyo flow
PM · QA: check SL/PT, desktop, mobile, dark mode, links, dynamic properties
PM · Send previews to client & kasey@luckandco.agency
PM · Update FigJam board with Klaviyo preview links
Client Approved
S · Activate flow in Klaviyo (+ A/B tests)
S · Deactivate old flow if replacing
S · Update CU task description ("Activated on [date]")
Activated ✓
Scenario C — Big changes
PM · Tag Strategist. Create subtask: "Review feedback + take next steps"
Client Reviewed
S · Review Figma comments, create copy/design edit subtasks as needed
Copy
(if copy edits needed)
C · Apply copy edits
Copy Review
S · Review copy edits
Design
D · Apply design edits in Figma (NO upload to Klaviyo yet)
Design Review
S · Review design edits in Figma
S · Tag PM when edits are ready
PM · Send updated Figma back to client
↩ Back to In Client Review — repeat until Scenario A or B
